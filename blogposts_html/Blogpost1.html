Canopy Height Models from LiDAR in Julia

Casey Ghilardi
December 9, 2019

Introduction

Julia is newer language designed to join the two worlds of human friendly syntax from higher level languages like Python with the performance of low level languages like C. A common pain point for newcomers is the lack of the the massive accompanying package ecosystem that languages such as Python or R have built up over the many years.

I ran into this myself when I was looking to play around with some LiDAR data. I was used to having packages such as lidR that do everything and the kitchen sink. While I can't quite match the full functionality present in other languages (yet), there is enough available to "self-help" and solve your own problems with some tinkering.

The rest of the post will walk through calculating canopy height models in Julia. Before I could do that, I realized I would need a ground filter to make the DEM. I chose the ground filter from Kraus and Pfeifer 1998. which is also the default ground filter used in the FUSION program. There is also a python implementation available as part of this package.

There is a nice write-up and visual for the filter process available here.

The basic steps to the filter are:

  1. Calculate an intermediate surface

  2. Calculate residuals between the points and the surface

  3. Update weights and recalculate in a loop

We'll follow the process the R version uses (the closest we can). There are some manual steps we'll need to do that FUSION takes care of behind the scenes.

1. Filter point cloud to bare earth points only.

The first step is to filter the entire point cloud to just ground points. The R version calls out to the groundfilter.exe function and returns a .las file.

The ground filter calculates a surface and then selects all points that fall within a tolerance parameter.

1a. Writing the function

I started with a very simple single cell version to help me understand the process before scaling up. This has an added benefit I'll touch on later.

We need a function that calculates weights based on some input vector.

function compute_weights(v_i::Array{Float64}, a, b, g, w)
    p_i = zeros(length(v_i))
    for i in 1:length(v_i)
        if v_i[i] <= g 
            p_i[i] = 1
        elseif v_i[i] > g && v_i[i] <= g + w
            p_i[i] = 1 / (1 + (a * (v_i[i] - g)^ b))
        else 
            p_i[i] = 0
        end
    end
    return p_i
end

compute_weights (generic function with 1 method)

#Some notes on Julia syntax if you've never seen it before: #- parameters after a ; are keyword arguments, we can update them otherwise they have a default value #- ::Array{Float64} tells us the input type, we don't have to put that but including it has performance and readability benefits.

Other than that, we are pre-allocating and array p_i and then iterating through it and updating it.

After that, the remaining functionality is to calculate a residual and then update the weights in a loop. 

function KrausPfeifer1998(points::Array{Float64}; a = 1.0,
                                                  b = 4.0,
                                                  g = -2.0,
                                                  w = 2.5, 
                                                  iterations = 5)
    z = points
    p_i = ones(Float64,length(points))
    for i in 1:iterations
        global surf = sum(z .* p_i)/sum(p_i)
        p_i = compute_weights(z .- surf, a, b, g, w)
    end
    return surf
end

KrausPfeifer1998 (generic function with 1 method)

2. Calulating for the entire point cloud

While having a working single cell example is nice, most point cloud are more than once cell. So now we have to make the function work across many cells. Thankfully, I found the PointCloudRasterizers package which takes care of most of this functionality for me.

1b. Filtering points

This is where the tolerance parameter I mentioned above comes back into play. Once the surface is calculated, points that fall within ± tolerance are designated as ground. 

tolerance = 1.0

#create a boolean function to pass to filter
ground(p) = p.Z .< dem_example + tolerance  && p.Z .> dem_example - tolerance

Calculate ground layer and canopy height

Now that we have the point cloud loaded, the spatial grid, and a working filter function we can put all the pieces together. I had mentioned an added benefit above and this is where it comes in.

The basic structure of PointCloudRasterizers applies a function cellwise and returns the resulting values as a raster. Since our ground filter is written to calculate the surface for a individual cell, we can use that as the reducer and we don't have to worry about creating an entire surface and iterating across it, we can just drop our function in as a reducer.

#Calculate the maxiumum heights for all cells. This is an example using a base Julia reducer.
canopy_example = reduce(inx, field=:Z, reducer = maximum)

It looks like our plot matches the article pretty closely. Our version has a "snowy" appearance where we have empty cells. After reading through FUSION's documentation (for the GridSurfaceCreate function specifically), I found

"Cells that contain no points are filled by interpolation using neighboring cells. Cells on the edge of the data coverage with no neighbors with valid elevations in one or more directions are flagged with NODATA values."

So FUSION is actually interpolating on the ground and canopy layers before subtracting. We'll need to do that explicitly before we subtract the layers.

Conclusions

We got a small tour of the Julia geo ecosystem, there are many packages out there that I did not touch on.

If we were working in different terrain, we may need to write a different ground filter.There is some early work implementing a progressive morphological filter (PMF) is available here. The filter we chose has some nice properties such as independent cell calculations that make it easy to implement (and write about) and works well for the forested dataset.

If you find yourself wanting to try Julia, but are frustrated by some lack of functionality my suggestion would be to dive right in and give it a shot. Hopefully you will be surprised how productive you can be.
